### All core devs (main protocol calls)

#### All core devs \- consensus (ACDC) [\#179](https://forkcast.org/calls/acdc/179)

* [Glamsterdam](https://forkcast.org/upgrade/glamsterdam) upgrade (targeting mid-2026):  
  * [glamsterdam-devnet-4](https://glamsterdam-devnet-4.ethpandaops.io/): investigating implementation bug, plan to relaunch once fixed as glamsterdam-devnet-5 with same spec  
  * ethPandaOps [disruptoor](https://github.com/ethpandaops/disruptoor#readme) (tool): trigger short & long range reorgs & forks in Kurtosis  
  * Engine API SSZ: [move & refactor proposal](https://github.com/ethereum/execution-apis/pull/793) selected, led by Marius van der Wijden, faster deprecation path as can drop JSON RPC at Hegotá upgrade     
  * Reviewed consensus layer EIPs currently considered for inclusion:  
    * [EIP8080](https://forkcast.org/eips/8080) Let exits use consolidation queue, plan to decline for inclusion, confirm at next ACDC  
    * [EIP8061](https://forkcast.org/eips/8061) Increase exit & consolidation churn, plan to schedule for inclusion as implemented in devnets, confirm at next ACDC  
    * [EIP8045](https://forkcast.org/eips/8045) Exclude slashed validators from proposing, review implementation complexity  
    * [EIP7688](https://forkcast.org/eips/7688) Forward compatible consensus data structures, review whether to include once devnets stable  
* [Hegotá](https://forkcast.org/upgrade/hegota) upgrade (targeting late-2026):  
  * Non-headliners proposed for inclusion:  
    * [EIP8148](https://forkcast.org/eips/8148) custom sweep threshold for validators  
* ACDC \#180 will trial 11am UTC start as more suitable for Australian timezones
